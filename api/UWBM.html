    1 UWBM

    UWBM(root=None, mode='w+', data_libs=None)

    HydroMT UWBM model class.

    This class provides a framework for setting up and running the Urban Water Balance Model (UWBM) as part of the HydroMT plugin. It manages the model region, geometries, landuse, forcing data, and configuration parameters.

    1.1 Methods

    Name Description
    setup_landuse Generate landuse map and associated tables for the model region.
    setup_pet_forcing Generate area-averaged, tabular reference evapotranspiration (PET) forcing for the model region.
    setup_precip_forcing Generate area-averaged, tabular precipitation forcing for geom.
    setup_project Setup the project geometry and basic configuration.

    1.1.1 setup_landuse

    UWBM.setup_landuse(source='osm', landuse_mapping_fn=None)

    Generate landuse map and associated tables for the model region.

    This function creates a polygon landuse map and a landuse table based on the provided base files and mapping. It also updates the configuration with landuse area and fraction statistics.

    1.1.1.1 Adds model layers

    • lu_map: polygon layer containing urban land use
    • lu_table: table containing urban land use surface areas [m2]

    1.1.1.2 Updates configuration

    • soiltype: soil type code according to UWB model documentation
    • croptype: crop type code according to UWB model documentation
    • landuse_area: surface area of landuse classes [m2]
    • landuse_frac: surface area fraction of landuse classes [-]
    • **tot_*_area**: total area of the UWB land use classes [m2]
    • **tot_*_frac**: total area fraction of the UWB land use classes [-]

    1.1.1.3 Parameters

    Name Type Description Default
    source str Source of landuse base files. Current default is “osm”. 'osm'
    landuse_mapping_fn str or None Name of the landuse mapping translation table. Default is None, in which case the default translation table for the source is used. None

    1.1.1.4 Returns

    Name Type Description
    None Updates the model’s geoms, landuse, and config objects in-place.

    1.1.1.5 Raises

    Name Type Description
    IOError If the provided source is invalid, the mapping file is missing, or the translation table is malformed.
    ValueError If the translation table columns are not of correct type or contain invalid classes.

    1.1.1.6 Notes

    For the “osm” source, the following layers are extracted from the data catalog: - osm_roads - osm_railways - osm_waterways - osm_buildings - osm_water

    The landuse table is generated from the landuse map and provides area and fraction for each reclassified landuse type. Total areas and fractions are calculated for main categories.

    1.1.1.7 Example

    model.setup_landuse(source=“osm”)

    1.1.2 setup_pet_forcing

    UWBM.setup_pet_forcing(temp_pet_fn='era5_hourly', pet_method='debruin')

    Generate area-averaged, tabular reference evapotranspiration (PET) forcing for the model region.

    This function adds a model layer for reference evapotranspiration:

    • pet: reference evapotranspiration [mm]

    1.1.2.1 Parameters

    Name Type Description Default
    temp_pet_fn str Name or path of the data source containing variables required to calculate temperature and reference evapotranspiration (default is ‘era5_hourly’). See data/forcing_sources.yml for available datasets. Required variables: - For temperature: [‘temp’] - For De Bruin PET: [‘temp’, ‘press_msl’, ‘kin’, ‘kout’] - For Makkink PET: [‘temp’, ‘press_msl’, ‘kin’] 'era5_hourly'
    pet_method str Method used to calculate reference evapotranspiration (default is ‘debruin’). Options: - ‘debruin’: calculates PET using the De Bruin method - ‘makkink’: calculates PET using the Makkink method 'debruin'

    1.1.2.2 Returns

    Name Type Description
    None Updates the model’s forcing object in-place with PET data.

    1.1.2.3 Raises

    Name Type Description
    ValueError If pet_method is not one of ‘debruin’ or ‘makkink’.

    1.1.2.4 Notes

    The PET data is resampled to match the model timestep. The output DataFrame includes both potential evapotranspiration (E_pot_OW) and reference grass evapotranspiration (Ref.grass). Attributes of the DataFrame include the data source (pet_fn) and the calculation method (pet_method).

    1.1.2.5 Example

    model.setup_pet_forcing(temp_pet_fn=“era5_hourly”, pet_method=“debruin”)

    1.1.4 setup_project

    UWBM.setup_project(region, name, t_start, t_end, ts=3600, crs=3857)

    Setup the project geometry and basic configuration.

    This function initializes the model region from a vector, sets the time range, timestep, coordinate reference system, and project name.

    1.1.4.1 Parameters

    Name Type Description Default
    region dict Dictionary describing the project region (e.g., geometry or boundary info). required
    name str Name of the project. required
    t_start str or datetime.datetime Start time of the simulation. If a string is provided, it will be converted to a datetime object. required
    t_end str or datetime.datetime End time of the simulation. If a string is provided, it will be converted to a datetime object. required
    ts int Timestep in seconds (default is 3600). Must be either 3600 (hourly) or 86400 (daily). 3600
    crs int Coordinate reference system for the project geometry (default is 3857). 3857

    1.1.4.2 Returns

    Name Type Description
    None Updates the model’s geoms and config objects in-place.

    1.1.4.3 Raises

    Name Type Description
    ValueError If ts is not 3600 or 86400.

    1.1.4.4 Notes

    The project geometry is processed through _parse_region and stored in the geoms object under the name “region”. The configuration settings such as start time, end time, timestep, and project name are stored in config.

    1.1.4.5 Example

    model.setup_project(region=my_region_dict, name=“TestProject”, t_start=“2025-01-01”, t_end=“2025-12-31”)
